**Отчет по лабораторной работе №2**
**1. Описание интеграции**
Какой источник данных выбрали
Для получения актуальных новостей использовался NewsAPI (https://newsapi.org).
Почему именно этот источник
NewsAPI предоставляет удобный и стандартизированный JSON-интерфейс.
Поддерживает категории: general, business, sports, science, что идеально подходит для нашего бота.
Простая регистрация и бесплатный тариф для небольших проектов.
Позволяет получать заголовки, описания и ссылки на оригинальные статьи, что облегчает интеграцию с Telegram.
**2. Промпт для LLM**
**Исходный промпт**
Интегрировать Telegram-бота с реальным источником новостей.
Показывать 5 последних новостей по выбранной категории.
Игнорировать статьи без валидной ссылки.
Сохранять избранные новости локально в JSON.
Итерации
Первая версия использовала статические ссылки (example.com).
Интеграция NewsAPI добавлена, но сначала ссылки иногда были некорректными.
Добавлена фильтрация некорректных URL и обработка ошибок сети/API.
Улучшен вывод новостей через HTML-разметку и команды для удобства пользователя.
**Финальный промпт**
Telegram-бот, который:
Получает 5 актуальных новостей с NewsAPI.
Отображает заголовки, описания и корректные ссылки.
Сохраняет избранное и фильтрует некорректные записи.
Корректно обрабатывает ошибки API и сети.
Использует понятные команды и меню для взаимодействия с пользователем.
**3. Реализация**
Как работает интеграция
При выборе категории бот вызывает функцию fetch_news_from_newsapi(category).
Функция делает GET-запрос к NewsAPI с параметрами category, country, pageSize и apiKey.
Ответ проверяется на ошибки:
отсутствие ключа NewsAPI
ошибки сети/HTTP
некорректный JSON
Новости фильтруются: удаляются записи без валидного URL и заголовка.
Выводятся пользователю через HTML-разметку с командами для сохранения в избранное.


**4. Трудности и решения**
**Проблемы**
Статические заглушки (example.com) вместо реальных ссылок.
Некоторые статьи приходили с пустым URL или некорректной ссылкой.
Возможные сбои сети или ошибки NewsAPI при запросах.
Пользователь мог попытаться сохранить новость до того, как открыл категорию.
**Решения**
Интегрировали NewsAPI вместо статических ссылок.
Добавлена фильтрация статей без URL и с некорректным URL.
Реализована обработка ошибок HTTP и сетевых исключений.
Сохраняем последние просмотренные новости в user_data, чтобы команды /save работали корректно.
**5. Выводы**
Что получилось хорошо
Интеграция с реальным источником новостей (NewsAPI) работает стабильно.
Новости корректно фильтруются и выводятся с HTML-разметкой.
Возможность сохранять избранное и просматривать список сохраненных статей.
Асинхронная обработка запросов не блокирует бот.
