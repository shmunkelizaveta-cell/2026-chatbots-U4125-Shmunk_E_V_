**Отчет по лабораторной работе №1**
**1. Описание задачи**
Цель лабораторной работы заключалась в создании Telegram-бота, который умеет показывать актуальные новости пользователю по категориям: политика, спорт, экономика и наука. Бот также должен позволять сохранять понравившиеся новости в избранное.
**Проблема, которую решает бот**
Современные пользователи часто хотят получать свежие новости в удобной форме. Бот решает проблему быстрого доступа к новостям без необходимости вручную посещать сайты новостных порталов. Пользователь получает новости прямо в Telegram, может просматривать категории и сохранять интересные статьи.
Почему выбрали именно эту задачу
Выбор пал на новостного бота, так как это практическая задача, включающая работу с API, обработку данных и взаимодействие с пользователем через интерфейс Telegram. Это хороший вариант для изучения базовых концепций Python, работы с внешними сервисами и хранения данных локально.
**2. Промпт для LLM**
Исходный промпт
Создать Telegram-бота на Python, который умеет показывать новости по категориям.
Новости должны приходить с реального API (NewsAPI).
Возможность сохранять новости в избранное в формате JSON.
Добавить базовое меню команд.
Итерации и улучшения
Первая версия использовала статические заглушки для новостей (example.com), что приводило к появлению тестовых ссылок.
Была интегрирована реальная NewsAPI для получения актуальных новостей.
Добавлена фильтрация некорректных URL, чтобы исключить пустые или фиктивные ссылки.
Улучшено отображение новостей через HTML-разметку с понятными кнопками и командами.
Добавлена обработка ошибок API и сети, чтобы бот не зависал при проблемах.
Финальный промпт
Telegram-бот, который:
Показывает 5 актуальных новостей по выбранной категории.
Поддерживает команды: /start, /help, /news, /politics, /sport, /economy, /science, /save, /favorites.
Сохраняет избранные новости в локальный JSON.
Корректно обрабатывает ошибки API и некорректные ссылки.
Отображает новости в читаемом формате с HTML-разметкой.
**3. Стек технологий**
Использованные библиотеки
python-telegram-bot — для работы с Telegram API и создания интерактивного бота. Выбрана за простоту интеграции и широкую документацию.
requests — для синхронных HTTP-запросов к NewsAPI. Простая и надёжная библиотека.
dotenv — для работы с переменными окружения (BOT_TOKEN, NEWSAPI_KEY, FAVORITES_FILE).
json — стандартная библиотека Python для хранения избранного в JSON формате.
asyncio — для асинхронной работы и не блокирования бота при сетевых запросах.
logging — для отслеживания ошибок и событий в боте.

**4. Трудности и решения**
Проблемы
При первом запуске бот выдавал ссылки вида example.com из-за заглушек.
Ошибки при обработке некорректного JSON или отсутствия ключа NewsAPI.
Некоторые пользователи не могли корректно сохранить новости, если не открывали категорию перед командой /save.
Некорректные ссылки могли появляться в выводе.
Решения
Интегрировали реальный NewsAPI, заменив статические заглушки.
Добавили проверку наличия NEWSAPI_KEY и обработку ошибок сети/HTTP.
Сохраняем последние показанные новости в user_data, чтобы /save работала корректно.
Фильтруем статьи без валидного URL, чтобы пользователь видел только рабочие ссылки.
